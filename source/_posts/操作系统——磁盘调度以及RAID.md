## 一、 磁盘调度算法

### 1.1 磁盘存取时间

磁盘完成一次读写操作所需的时间由三部分组成：

$$
T_a = T_s + \frac{1}{2r} + \frac{b}{rN}
$$

- **$Ts$ (Seek Time)**：寻道时间，磁头臂移动到指定磁道的时间。**调度算法的核心优化目标**。
- **$1/2r$**：旋转延迟，平均需要等待半圈扇区旋转到磁头下方。
- **$b/rN$**：传送时间，实际读写数据的时间。

### 1.2 调度策略分类

| 调度类型 | 核心算法 | 描述 | 特点与问题 |
| :--- | :--- | :--- | :--- |
| **移臂调度** | **FCFS** (先来先服务) | 按请求到达的先后顺序处理。 | 简单、公平；但移臂距离大，性能差。 |
| | **SSTF** (最短寻道时间优先) | 选择距离当前磁头最近的请求。 | 寻道性能好，但可能导致**饥饿**（远处请求长期得不到服务）。 |
| | **SCAN** (电梯调度) | 磁头单向移动，沿途服务请求，到尽头再折返。 | 避免饥饿，兼顾公平与性能。 |
| | **C-SCAN** (循环扫描) | 单向扫描，到尽头后直接返回起始端，再次扫描。 | 对所有磁道响应时间更均匀。 |
| | **LOOK** | SCAN的改进，磁头移动到**最远请求**后折返，不必到尽头。 | 减少不必要的移动。 |
| | **C-LOOK** | C-SCAN的改进，磁头移动到最远请求后直接返回。 | 结合了C-SCAN的均匀性与LOOK的效率。 |
| **旋转调度** | **循环排序** | 对同一柱面的多个请求，按扇区旋转顺序排序。 | 避免磁头反复等待旋转，在最少旋转圈数内完成。 |
| | **优化分布** | 通过调整数据在磁道上的物理排列（如交替编号），使得在读完一个数据后，下一个要读的数据恰好旋转到磁头下。 | 极大减少旋转延迟。 |

### 1.3 经典算法示例对比 (柱面0-199，起始点53)

| 算法 | 访问顺序 | 总移动柱面数 |
| :--- | :--- | :--- |
| **FCFS** | 53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67 | 640 |
| **SSTF** | 53 → 65 → 67 → 37 → 14 → 98 → 122 → 124 → 183 | 236 |
| **SCAN (向0移动)** | 53 → 37 → 14 → 0 → 65 → 67 → 98 → 122 → 124 → 183 | 208 |
| **C-SCAN (向0移动)** | 53 → 37 → 14 → 0 → 199 → 183 → 124 → 122 → 98 → 67 → 65 | 208 + 199 = 407 |

## 二、 RAID (磁盘冗余阵列)

### 2.1 基本概念

- **定义**：将多个独立磁盘组合成一个逻辑磁盘，通过**并行存取**提高性能，通过**冗余校验**提高可靠性。
- **设计目标**：解决CPU性能增长与磁盘I/O性能增长不平衡的问题。
- **关键术语**：
    - **Striping (条带化)**：将数据分块，交叉存放到多个磁盘，提高并行I/O。
    - **Redundancy (冗余)**：通过校验码或镜像来保证数据在部分磁盘损坏时不丢失。

### 2.2 常见RAID级别对比

| 级别 | 数据分布方式 | 冗余方式 | 空间利用率 | 读性能 | 写性能 | 容错能力 (可坏盘数) | 适用场景 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **RAID 0** | 条带化 (Striping) | 无 | 100% (N块盘 → N倍) | 高 (N倍) | 高 (N倍) | 0 | 追求速度，不要求安全（如临时缓存、游戏） |
| **RAID 1** | 镜像 (Mirroring) | 完全复制 | 50% (N/2) | 高 (N倍) | 最慢的磁盘 | N-1 | 高安全性（如操作系统、关键数据） |
| **RAID 2** | 位级条带化 | 海明码 | (n)/(n+k) | 较高 | 低 (受校验盘限制) | 1 | 顺序数据访问，成本高，现已罕见 |
| **RAID 3** | 字节/位级条带化 | 单块奇偶校验盘 | (N-1)/N | N-1倍 | 低 (校验盘瓶颈) | 1 | 大容量顺序数据（如视频流、科学计算） |
| **RAID 4** | 块级条带化 | 单块奇偶校验盘 | (N-1)/N | 高 (N-1倍) | 低 (校验盘瓶颈) | 1 | 随机读多、写少的数据 |
| **RAID 5** | 块级条带化 | 分布式奇偶校验 | (N-1)/N | 高 (N倍) | 中等 (写惩罚) | 1 | **综合性能最佳**（如文件服务器、数据库） |
| **RAID 6** | 块级条带化 | 双重分布式奇偶校验 | (N-2)/N | 高 (N倍) | 较低 (写惩罚更大) | 2 | 高安全性需求（如金融、档案存储） |
| **RAID 10** | 先镜像(RAID1)，再条带(RAID0) | 镜像 | 50% | 高 | 高 | N-1 | 既追求速度又要求安全（如高性能数据库） |

### 2.3 关键RAID级别特性补充

- **RAID 0 vs. RAID 5**：RAID 0 写性能略好于 RAID 5，但无容错能力。
- **RAID 3 vs. RAID 4**：RAID 3 适合**大块顺序数据**（如视频），RAID 4 适合**小块随机数据**（如文件）。
- **RAID 5 vs. RAID 6**：RAID 6 提供双重校验，可容忍**两块盘同时损坏**，安全性更高，但写性能略低，空间利用率稍低。
- **RAID 10 (RAID 1+0)**：结合了RAID 1的高可靠性和RAID 0的高性能，**既快又安全**，但成本高（利用率50%）。

### 2.4 其他补充内容

- **Disk Cache (磁盘缓存)**：利用主存中开辟的缓冲区缓存磁盘扇区数据，减少实际磁盘I/O。
- **SSD (固态硬盘)**：当前主流趋势，无机械部件，速度远超机械硬盘（顺序读速可达7000MB/s以上，如NVMe SSD）。

## 三、 核心公式与计算技巧（考试/面试重点）

### 3.1 磁盘时间计算

- **总存取时间** = 寻道时间 + 旋转延迟 + 传送时间
- **旋转延迟** = 旋转半圈时间 = $(60 / RPM) / 2$ (RPM为转速，如7200 rpm)

### 3.2 RAID容量与空间利用率

- **RAID 0**：总容量 = N × 单盘容量
- **RAID 1**：总容量 = (N/2) × 单盘容量
- **RAID 3/4/5**：总容量 = (N-1) × 单盘容量
- **RAID 6**：总容量 = (N-2) × 单盘容量
- **RAID 10**：总容量 = (N/2) × 单盘容量 (N为偶数)

### 3.3 常用调度策略选择

- **追求公平性**：FCFS
- **追求最低平均寻道时间**：SSTF (但有饥饿风险)
- **避免饥饿、兼顾公平与性能**：SCAN 或 LOOK
- **对磁道响应时间要求均匀**：C-SCAN 或 C-LOOK
